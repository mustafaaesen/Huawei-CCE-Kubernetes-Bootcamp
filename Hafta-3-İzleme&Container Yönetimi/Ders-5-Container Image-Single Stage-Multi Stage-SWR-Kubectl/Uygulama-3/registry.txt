LOCAL'E ÖZEL REGISTRY OLUŞTURMA

Docker standart API 'ından registry oluşturup image'ları oraya push etmek

Aşağıdaki komutlar takip edilebilir.


---> docker run -d -p 5000:5000 --name local-registry registry:2
(2 sürümlü registry apı'nı kullanrak "local-registry" adında 5000'i 5000' e yönlendiren registry oluşturma)
Localde çalışan bir registry var artık

---> docker tag uyg1:v1 localhost:5000/my-single-stage-app:v1.0.0
(daha önce oluşturulmuş olan image'lardan birini hazırlama adını localhosttan sonraki yazana çevirir ) 

--->docker push localhost:5000/my-single-stage-app:v1.0.0
(oluşturulan yeni image'ı push etme)

Artık bu image localde çalışan bir registry içerisine alınmış oldu


---> docker pull localhost:5000/my-single-stage-app:v1.0.0
(registry'den image'ı çekmek)

ÖZET: image'ların silinme kayıp durumuna karşı bağımsız bir yerde kayıtlı tutmanın önemi olan DockerHub
Local bilgisyarda deneyimlenmiş oldu.

curl -X GET http://localhost:5000/v2/_catalog
(registry içerisinde ne olduğunun listesi)

curl -X GET http://localhost:5000/v2/uyg1/my-single-stage-app/tag/list
(şmage hakkında bilgi alma)


registry mantığını anlamak ve simüle etmek için local'de denenmiştir
çalışan projelerde local bir bilgisyarda registry tutmak çok mantıklı değildir.
